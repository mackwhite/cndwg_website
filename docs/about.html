Data Agreement

The following agreement has benefited greatly from and contains significant portions of templates/information derived from peer-reviewed literature (i.e., Cheruvelil et al., 2014; Boland et al., 2017) and formalized data policies (i.e., Soil Data Harmonization & Synthesis, Arctic Data Center). Each of these resources were used in consideration of the LTER data requirements for synthesis working groups. This policy is a living document that may be revised to reflect changes in working group participants, datasets, and project goals while ensuring compliance with the LTER derived data requirements per our funding agreements.

It is well established that collaborative synthesis research elicits a deeper and more holistic understanding of our natural world. In this synthesis effort, we will use publicly available, data published in peer-reviewed articles, and data from unpublished sources. Detailed metadata and a thorough understanding of each dataset will be necessary to link them across sites in a single database. As such, the cooperation and collaboration of scientists with intimate understandings of their respective datasets/sites will be critical to generating accurate research products. Furthermore, we believe that the working group will greatly benefit from the intellectual contributions of a team of scientists with diverse research backgrounds.

This project will synthesize multiple data sources. This includes, but is not limited to data published in publicly available databases, data which is publicly available but are protected via prior data sharing agreements, and those which are not publicly available. The primary purpose of this document is to facilitate clear understanding of how data will be (1) used and (2) shared. It is expected that perspectives and technologies concerning data will change through conversations with working group participants and any revisions to the data policy will promptly be updated in the shared google drive folder until the development of a project-specific website is completed in GitHub. Following, updates to the data policy will be reflected on the website and all working group participants notified via email. We look to the SOils DAta Harmonization (SoDaH) & Synthesis working group to differentiate data at different tiers of manipulation (see below).

*modified from SoDaH working group workflow figure

Raw data (Tier 0) in their native format, alongside metadata will be housed in the working groups google drive and R scripts for data wrangling will be archived in  LTER working group GitHub repository to facilitate data harmonization (Tier 1), aggregation, and dataset end-products (Tier 2). The derived database will be managed by the database manager, Li Kui (UCSB). The database manager (Li Kui; lkui@ucsb.edu) and principal investigators (Joseph Peters; jpeters@ucsb.edu, Mack White; mwhite@fiu.edu, Bradley Strickland; bstrickland@vims.edu, Jennifer Rehage; rehagej@fiu.edu) are the only individuals with authority to modify the Tier 2 data and will only do so if necessary. Any modifications to the Tier 2 will be summarized in GitHub and available for review by all working group participants and collaborators. Working group participants and collaborators are not allowed to share or make available in any form any data contained within the database during the duration of the project, unless discussed with the entirety of the working group.

Data Package

We anticipate many participants contributing data. All data providers will be invited to be coauthors on a data package (i.e., final database) published through the Environmental Data Initiative (EDI). However, we strongly suggest that each data provider publish their individual dataset through EDI. This will ensure that each dataset has the appropriate meta-data and can be properly cited. Remember that providing data is only one of at least two required contributions for co-authorship on peer-reviewed journal articles prepared by the group (see authorship agreement). We expect each data provider to collate data, write metadata, provide appropriate attribution information, and review and approve the final data package.

GitHub

All analyses will be conducted in R. Some data reports may be generated through Rmarkdown. All code for analysis and visualization will be shared on the project GitHub repository. Upon publication of a manuscript within the working group, relevant code will be archived using Zenodo.
